\section{Системийн үйл ажиллагааны тухай дэлгэрэнгүй}

\subsection*{Системийн хэрэглэгчийн сценари}

\subsubsection{Сценари 1: Хэрэглэгч (Оюутан, Суралцагч)}
Хэрэглэгч өөрийн бүртгэлээр системд нэвтэрч ормогц өмнө нь үүсгэсэн бүх хичээлүүд жагсаалтаар харагдана. Хичээл бүрийн хажууд үзсэн статус (Completed / In progress), ахицын хувь, үүсгэсэн огноо харагдах ба хэрэглэгч аль хичээлээ үргэлжлүүлэн судлахаа сонгох боломжтой. Шинэ хичээл үүсгэх товчийг дарж, шууд хоосон хичээл эхлүүлэх, эсвэл номын файл (PDF болон Зураг) оруулж хичээл үүсгэх боломжтой. 

Хэрэглэгч хичээлийн номыг PDF файл хэлбэрээр оруулахад PDF Parser ашиглан. Үүний дараа тухайн материалыг хичээл $\rightarrow$ бүлэг $\rightarrow$ дэд сэдэв гэсэн бүтэц рүү автоматаар задалж, хэрэглэгч аль сэдвүүдийг хичээлдээ оруулахаа сонгоно.

Хичээл дотор шинэ тэмдэглэл үүсгэх үед файл хавсаргах боломжтой. Тэмдэглэлийн материал илгээгдмэгц AI Model API (Chimege OCR, Egune) ашиглан сэдвийн хураангуй, флаш карт, түлхүүр ойлголтууд болон тестийн асуултуудыг автоматаар үүсгэнэ. Энэ боловсруулалтын явц дэлгэцэд хувиар харагдана. Хэрэглэгч AI үүсгэсэн контентыг өөрийн хүссэнээр засварлах, нэмэлт чөлөөт текст бичих боломжтой. 

Суралцах явцдаа ойлгохгүй зүйл гарвал хичээлийн Chatbot хэсэг рүү орж, тухайн хичээлийн агуулгад тулгуурлан AI-аас асуулт асууж, нэмэлт тайлбар авна. Шалгалт өгөх үед систем боловсруулсан асуултуудаар тест авч, дүгнэнэ. Хэрэв үр дүн 90\%-иас дээш байвал тухайн сэдвийг "Completed" буюу үзэж дууссанаар тэмдэглэж, ахицын хувь автоматаар шинэчлэгдэнэ. Мөн хэрэглэгч өөрийн үүсгэсэн сайн хичээлээ бусадтай хуваалцахыг хүсвэл "Public" төлөвт шилжүүлж, үүссэн линкийг бусдад илгээх боломжтой.

\subsubsection{Сценари 2: Админ}
Админ эрхтэй хэрэглэгч системд нэвтэрмэгц удирдлагын самбар (Admin Dashboard) руу шилжинэ. Энд системийн нийт хэрэглэгчид, идэвхтэй хичээлүүд болон AI боловсруулалтын статистик харагдана. Админ хэрэглэгчдийн жагсаалт руу орж бүртгэлийн мэдээлэл, үүсгэсэн хичээлийн тоо зэргийг хянана. Мөн хэрэглэгчийн хичээлүүдийг нээж, хичээлийн бүтэц, тэмдэглэлүүд, AI болон OCR боловсруулалтын үр дүнг шалгах боломжтой. Хэрэв тэмдэглэлийн материал боловсруулалтад алдаа гарсан, эсвэл AI буруу контент үүсгэсэн байвал админ тухайн материалыг системээс "Дахин боловсруулах (Reprocess)" үйлдэл хийж контентыг шинэчлэх, шаардлагатай бол зохисгүй агуулгыг устгах эрхтэй байна.

\section{Системийг ашиглах хэрэглэгчид}
Системийн зорилго болон хэрэглээний сценарид үндэслэн системийг ашиглах хэрэглэгчид (Actors)-ийг дараах байдлаар тодорхойлов:
\begin{enumerate}
    \item \textbf{Хэрэглэгч (Оюутан/Багш):} Системийн үндсэн хэрэглэгч. Хичээл үүсгэх, файл оруулах, AI-аас тусламж авах, шалгалт өгөх, өөрийн ахиц дэвшлээ хянах, хичээлээ бусадтай хуваалцах (Public линк) бүрэн эрхтэй.
    \item \textbf{Админ (Системийн зохицуулагч):} Системийн хэвийн үйл ажиллагааг хянах, хэрэглэгчдийн датаг удирдах, алдаатай боловсруулалтыг дахин эхлүүлэх, нийт статистик мэдээллийг харах эрх бүхий хэрэглэгч.
    \item \textbf{Гадаад систем (AI \& OCR API):} Системтэй шууд харилцаж өгөгдөл боловсруулах хиймэл оюуны модел (Egune LLM) болон зураг танигч модел (Chimege OCR).
\end{enumerate}

\section{Функцийн шаардлага}
Системийн үйл ажиллагааны үндсэн функцүүдийг гүйцэтгэх үүргийнх нь дагуу логик модулиудад хуваан тодорхойлсон байдал. \\

\textbf{1. Хэрэглэгчийн бүртгэл болон удирдлага}
\begin{itemize}
    \item Бүртгэл үүсгэх, нэвтрэх, системээс гарах, нууц үг сэргээх.
    \item Өөрийн профайл мэдээллийг харах, засварлах.
    \item Нүүр хуудсанд өөрийн бүх хичээлийг жагсаалтаар харах, хайх, шүүх (нэр, огноо, статус).
    \item Хэрэглэгчийн хяналтын самбараас шалгалтын дундаж оноо, амжилтын хувь, сүүлд үзсэн хичээлээ харах.
\end{itemize}

\textbf{2. Хичээл болон файл боловсруулалт}
\begin{itemize}
    \item Шинэ хичээл хоосон хэлбэрээр үүсгэх.
    \item Номын файл (PDF) болон зурган файл (Сканнердсан зураг) оруулж хичээл үүсгэх.
    \item \textbf{[OCR Функц]} Оруулсан материалыг OCR технологи ашиглан текстийг таних.
    \item Оруулсан номыг PDF Parser-оор боловсруулж хичээл $\rightarrow$ бүлэг $\rightarrow$ дэд сэдэв гэсэн бүтэц автоматаар үүсгэх.
    \item Боловсруулсан бүтэц дотроос хичээлдээ оруулах сэдвүүдийг сонгох.
\end{itemize}

\textbf{3. AI Контент үүсгэлт болон Тэмдэглэл}
\begin{itemize}
    \item Хичээл дотор шинэ тэмдэглэл үүсгэж, файл хавсаргах.
    \item Тэмдэглэлийн материал оруулмагц OCR модел ашиглан текст танилт хийн үр дүнг AI Model (LLM) руу илгээж, \textbf{хураангуй, түлхүүр үг, флаш карт, тестийн асуултууд} автоматаар үүсгүүлэх.
    \item Хэрэглэгч AI-ийн үүсгэсэн контент, флаш картыг нэмэх, засварлах, устгах, чөлөөт текст бичих.
    \item Хичээлийн Chatbot-д тухайн хичээлийн агуулгад тулгуурласан асуулт тавьж хариулт авах.
\end{itemize}

\textbf{4. Суралцах явц болон Шалгалт}
\begin{itemize}
    \item Хичээл бүрийн үзсэн статус, ахицын хувийг хянах.
    \item AI-ийн үүсгэсэн асуултаар шалгалт өгөх, дүнгээ (зөв/буруу) харах.
    \item Шалгалтын үр дүн 90\%-иас дээш бол тэмдэглэлийг автоматаар "Үзэж дуусгасан" төлөвт шилжүүлэх.
    \item Хичээлийн бүх тэмдэглэлийг дуусгасан үед хичээл "Үзэж дуусгасан" болгох.
\end{itemize}

\textbf{5. Хичээл хуваалцах болон Админы удирдлага}
\begin{itemize}
    \item Хичээлийг Private эсвэл Public болгож тохируулах.
    \item Public хичээлд унших эрхтэй хуваалцах (share) линк үүсгэх.
    \item Админ хэрэглэгчдийн жагсаалт, хичээлүүдийн бүтэц, AI боловсруулалтын үр дүнг хянах.
    \item Админ алдаатай тэмдэглэлийг дахин боловсруулах үйлдэл хийх.
    \item Админ системийн тайлан, статистик харах.
\end{itemize}

\section{Функцийн бус шаардлага}
\begin{enumerate}
    \item \textbf{Гүйцэтгэл ба Хурд (Performance):} 
    \begin{itemize}
        \item Систем хэрэглэгчийн энгийн хүсэлтэд 3 секундээс бага хугацаанд хариу өгнө.
        \item AI болон OCR боловсруулалтын үр дүнд хураангуй, флаш карт, тест үүсгэх хүнд үйл явц 30 секундээс хэтрэхгүй хугацаанд дууссан байна.
    \end{itemize}  
    \item \textbf{Аюулгүй байдал:}
    \begin{itemize}
        \item Хэрэглэгчийн нууц үгийг шифрлэн хадгална.
        \item Систем нь HTTPS протокол ашиглан бүх өгөгдөл дамжуулалтыг шифрлэнэ.
    \end{itemize}  
    \item \textbf{Хандалтын хязгаарлалт:} 
    \begin{itemize}
        \item Хэрэглэгчийн өгөгдөлд зөвхөн тухайн хэрэглэгч өөрөө хандах эрхтэй байх.
        \item Role-based authentication хэрэгжүүлж, хэрэглэгч болон админы эрхийг хатуу ялгана.
    \end{itemize}
    \item \textbf{Хяналт ба Бүртгэл:} 
    \begin{itemize}
        \item Системийн үйл ажиллагааны лог бүртгэл тодорхой байж, алдаа илрүүлэхэд хялбар байна.
        \item Огноо/цагийг хэрэглэгчийн бүсийн цагийн бүсээр зөв хувиргаж харуулна.
    \end{itemize}  
    \item \textbf{Хэрэглэхэд хялбар байдал:}
    \begin{itemize}
        \item Вэб интерфэйс нь responsive дизайнтай байж, desktop болон mobile төхөөрөмж дээр адилхан саадгүй ажиллана.
        \item Алдааны болон анхааруулгын мэдэгдлүүд хэрэглэгчид ойлгомжтой байдлаар харагдана.
    \end{itemize}
\end{enumerate}

\section{Ажлын хүрээ}
Системийн шаардлагуудыг хөгжүүлэлтийн үе шат, чухал байдлаар нь MoSCoW аргачлалаар дараах байдлаар эрэмбэллээ.

\subsection{Must Have (Зайлшгүй байх шаардлагатай)}
\begin{itemize}
    \item Хэрэглэгчийн бүртгэл болон нэвтрэх систем.
    \item PDF болон Зураг (OCR) файл системд оруулах, текст болгон задлах.
    \item AI ашиглан автоматаар сэдвийн хураангуй, флаш карт, сорил үүсгэх.
    \item Шалгалт өгөх, оноо тооцох, хичээлийн ахицын хувийг хянах.
\end{itemize}

\subsection{Should Have (Байх ёстой чухал функцүүд)}
\begin{itemize}
    \item Хичээлийн агуулгатай холбогдож асуултад хариулах Chatbot.
    \item Өөрийн хичээлийг бусдад линкээр хуваалцах (Public/Private тохиргоо).
    \item Админ удирдлагын самбар болон дахин боловсруулах (Reprocess) функц.
\end{itemize}

\subsection{Could Have (Байвал сайн нэмэлт боломжууд)}
\begin{itemize}
    \item Тэмдэглэл дотор нэмэлтээр аудио бичлэг оруулах.
    \item Хэрэглэгчийн сургалтын нарийвчилсан статистик, график дүрслэл.
\end{itemize}

\subsection{Won't Have (Энэ удаагийн хөгжүүлэлтэд багтахгүй)}
\begin{itemize}
    \item Гар утасны бие даасан (Native) аппликейшн хөгжүүлэх (зөвхөн Responsive вэб байна).
    \item Олон оюутан нэг зэрэг шалгалт өгч өрсөлдөх горим (Multiplayer quiz).
    \item Төлбөр тооцооны систем нэгтгэх.
\end{itemize}

\section{Шинжилгээний дарааллын диаграм}
Хэрэглэгч файл оруулж, систем түүнийг хэрхэн хүлээж авч OCR болон AI серверүүдтэй харилцан өгөгдөл буцааж буй хугацааны дарааллыг Дарааллын диаграмаар харуулав. Энэхүү диаграм нь Frontend $\rightarrow$ Backend (Golang) $\rightarrow$ OCR Service $\rightarrow$ AI Model $\rightarrow$ Database гэсэн дарааллаар мэдээлэл хэрхэн шилжиж байгааг илэрхийлнэ.

\section{Үйл ажиллагааны диаграм}
Оюутан хичээл үүсгэхээс эхлээд, материалаа унших, шалгалт өгөх, хэрэв шалгалтад унавал дахин унших, тэнцвэл "Completed" төлөвт шилжиж дуусах хүртэлх хэрэглэгчийн алхам бүрийн шийдвэр гаргах урсгалыг (Control flow) үйл ажиллагааны диаграмаар үзүүллээ.

\section{Бүлгийн дүгнэлт}
Хоёрдугаар бүлгийн хүрээнд хичээлийн ухаалаг тэмдэглэлийн системийн үйл ажиллагааны бодит хэрэгцээнд тулгуурлан хэрэглэгчийн болон админы сценарийг боловсруулж, системийг ашиглах хэрэглэгчдийг нарийвчлан тодорхойллоо. Системийн функцийн шаардлагуудыг логик модулиудад бүлэглэн гаргаснаар хөгжүүлэлт хийх үе шат (MoSCoW аргачлал) илүү тодорхой болсон.

Ялангуяа системд тавигдах онцлог шаардлага болох зураг таних (OCR) болон AI боловсруулалтыг Юзкейс диаграм болон Юзкейсийн тодорхойлолтоор нарийвчлан задалж үзүүлэв. Мөн системийн дотоод архитектур, объектуудын харилцан хамаарал, мэдээллийн урсгалыг Класс, Дараалал, Үйл ажиллагааны UML диаграмуудаар дүрслэн харуулсан нь дараагийн бүлэгт системийн зохиомж болон өгөгдлийн сангийн бүтцийг боловсруулах үндсэн суурь баримт бичиг (System Requirements Specification) болж чадлаа.
